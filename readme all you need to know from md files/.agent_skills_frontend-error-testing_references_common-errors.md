# Common Frontend Errors Reference

## JavaScript Errors

### TypeError
- **Symptom**: `TypeError: x is not a function` or `TypeError: Cannot read properties of null (reading 'y')`.
- **Diagnosis**: Use `evaluate_script` to check the value of the parent object.
- **Common Fix**: Add optional chaining (`?.`) or proper null checks.

### ReferenceError
- **Symptom**: `ReferenceError: x is not defined`.
- **Diagnosis**: Check `list_console_messages` for the exact line.
- **Common Fix**: Verify imports or global variable initialization.

## Network & Connectivity

### CORS Issues
- **Symptom**: `Access-Control-Allow-Origin` errors in console.
- **Diagnosis**: Inspect headers with `get_network_request`.
- **Action**: Verify backend CORS configuration or use an appropriate proxy.

### WebSocket (WS) Failures
- **Symptom**: `NS_ERROR_WEBSOCKET_CONNECTION_REFUSED`.
- **Diagnosis**: Use `list_network_requests` and filter by `Type: websocket`. Check the URL construction in the source code.

## Framework Specifics (Heuristics)

- **React**: Check for "Minified React Error" (usually requires looking at the source map or dev build logs).
- **Vite/Webpack**: Check for HMR (Hot Module Replacement) errors in the console if the page doesn't update.
